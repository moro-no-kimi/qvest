# Fulton County Reading Lift Pilot - Proof of Concept

A hybrid AI-powered book recommendation system for middle school libraries, designed to increase student reading engagement while keeping librarians in control.

## System Overview

This proof of concept demonstrates:

1. **Student Experience**: Personalized book recommendations with plain-English explanations
2. **Librarian Workspace**: Review, approve, and manage recommendations 
3. **District Dashboard**: Analytics and pilot performance metrics
4. **Hybrid AI Engine**: Combines collaborative filtering, content-based recommendations, and policy guardrails

## Quick Start

1. **Setup Environment**
   ```bash
   cd claudecode
   pip install -r requirements.txt
   python setup.py
   ```

2. **Start the API Server**
   ```bash
   python api/main.py
   ```

3. **Launch Interfaces** (in separate terminals)
   ```bash
   # Student interface
   streamlit run ui/student_app.py --server.port 8501

   # Librarian workspace  
   streamlit run ui/librarian_app.py --server.port 8502

   # District dashboard
   streamlit run ui/district_dashboard.py --server.port 8503
   ```

4. **Access Applications**
   - Student App: http://localhost:8501
   - Librarian App: http://localhost:8502
   - District Dashboard: http://localhost:8503
   - API Docs: http://localhost:8000/docs

## Key Features Demonstrated

### Student Experience
- **Personalized Recommendations**: Top 5 books based on reading history
- **Clear Explanations**: "Because you liked X, you might enjoy Y"
- **Simple Actions**: Save books of interest
- **Grade-Appropriate**: Only shows books suitable for student's grade level

### Librarian Oversight
- **Review Queue**: See all pending recommendations
- **Approval Actions**: Approve, reject, pin, or replace recommendations
- **Student Lookup**: Search by student or homeroom
- **Provenance**: Understand why each book was recommended

### District Analytics
- **Pilot Health**: Adoption rates, approval rates, checkout lift
- **School Comparison**: Performance across pilot schools
- **Trend Analysis**: Weekly usage and engagement patterns
- **Quality Signals**: Librarian action rates and system trust metrics

### AI Recommendation Engine
- **Collaborative Filtering**: "Students like you also borrowed..."
- **Content Similarity**: Genre, theme, and metadata matching  
- **Cold Start Handling**: Popular books for new/inactive students
- **Policy Guardrails**: Grade-level and availability filtering

## Data Model

### Books
- Title, Author, Genre, Grade Range, Series
- Lexile Level, Description, Availability
- Generated from realistic middle-grade literature patterns

### Students  
- Grade (6-8), School, Reading Level
- De-identified IDs for privacy
- Distributed across 5 pilot schools

### Checkout History
- 8,000+ synthetic checkout events
- Realistic borrowing patterns and preferences
- 2-year historical window

### Recommendations
- Generated by hybrid AI engine
- Tracked through approval workflow
- Linked to explanations and provenance

## Technical Validation

The system demonstrates:

1. **Collaborative Filtering**: Users with similar checkout patterns get similar recommendations
2. **Content-Based Fallback**: New students get genre/theme matches  
3. **Policy Compliance**: Grade-inappropriate books are filtered out
4. **Human Oversight**: Librarian actions change what students see
5. **Scalable Architecture**: Batch processing, API-driven, database-backed

## Pilot Simulation

The proof of concept includes:
- **5 middle schools** with realistic enrollment
- **500 books** covering diverse genres and grade levels  
- **1,000 students** with varied reading histories
- **8,000 checkouts** showing authentic borrowing patterns
- **1,500+ recommendations** at various approval stages

## Privacy & Governance

- **De-identified student IDs**: No PII in recommendation pipeline
- **Role-based access**: Students see only their recommendations
- **Audit trails**: All librarian actions logged
- **Local processing**: No student data sent to external LLMs

## Production Readiness Path

This POC establishes the foundation for:
- **SSO Integration**: ClassLink + Microsoft Entra ID
- **Real Data Pipeline**: Destiny Library Manager exports  
- **Production Infrastructure**: PostgreSQL, Redis, container deployment
- **Enhanced Analytics**: A/B testing, statistical significance
